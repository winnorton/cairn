# Plan: v0.7 — Collaboration Skills + Taxonomy

**Status:** Shipped
**Date:** 2026-04-24
**Driving critique:** Issue #16 — systematic transcript analysis surfaced three more human-interaction patterns worth formalizing, plus a conceptual layer distinguishing collaboration skills from maintenance skills.

## Goal

Add three collaboration skills to cairn and introduce the taxonomy that separates them
from the existing maintenance skills. Both came from observational analysis of the cairn
build session transcript (60 human messages) — not theoretical gap analysis.

## Why

`/reframe` was cairn's first skill designed to serve the *human* rather than the
habitat. It landed because the human had done something the agent couldn't. That
origin suggested a question: what other moves does the human make that the agent
can't, and could any be formalized?

A research agent in a parallel Cowork session analyzed the full transcript and
identified 8 interaction patterns. Three pass the formalization test that produced
`/reframe`:

- Cross-session relay (7+ instances, highest cognitive load)
- Timing judgment (decisive overrides on "cheap now / expensive later")
- User advocacy (reports on how other agents use cairn)

One more pattern (triage / rapid prioritization) is a convention within `/plan` rather
than a standalone skill.

## Design

### New skills

**`/bridge` — cross-session relay.** Structure the handoff when the user carries
output between parallel agent sessions. Identify the source, extract the key finding,
ask assess/integrate/act. For outgoing bridges (this session's output relevant
elsewhere), flag proactively. Role: `scaffolding`. Tier: `grow`.

**`/advocate` — end-user perspective.** Before shipping user-facing output, simulate
how a first-time user would experience it. Name the user, simulate the cold read,
surface specific friction points, propose mitigations. Role: `scaffolding`. Tier: `grow`.

### Plan enhancement (not a new skill)

**Timing awareness in `/plan`.** New step 5: if any plan step has a lifecycle-dependent
cost curve (cheap now, expensive later — or vice versa), name it in one line. The agent
defaults to conservative; the human has the window-awareness the agent doesn't. Naming
the dimension gives the human an opening.

The `/tempo` pattern identified in the research paper becomes an inline step rather
than its own skill. The paper itself suggests this ("might fold into /plan").

### Taxonomy

Cairn's skills now fall into two categories:

**Maintenance skills** (service the habitat):
`tour`, `reflect`, `plan`, `prune`, `audit`, `feedback`

**Collaboration skills** (service the human-agent pair):
`reframe`, `bridge`, `advocate`

The maintenance skills existed from v0.1 and were identified by gap analysis — what the
agent noticed it needed. The collaboration skills came from observation — what the
human actually does in the collaboration.

`skills/README.md` and the main `README.md` now explain this split with the design
principle it implies: **observe the collaboration first, then package what you see.**
Skills designed from observation solve problems that exist; skills designed from theory
often solve problems that don't.

## What was skipped

- **`/tempo` as standalone skill.** Folded into `/plan` per the research paper's own
  recommendation. Reconsider if the inline step proves insufficient.
- **`/triage` as standalone skill.** Same reasoning — it's a formatting discipline
  within `/plan`, not a distinct capability.
- **Agent-side trigger reliability testing.** The collaboration skills have fuzzier
  triggers than maintenance skills. Whether agents detect "user is bridging" vs.
  "user is asking a question" reliably is an open question the research paper flags;
  needs real usage to answer.

## Outcome

Shipped as v0.7.0. Two new skills, one `/plan` enhancement, taxonomy in README and
skills/README. Research artifacts (`collaboration-skills.md` + `human-interaction-patterns.md`)
added to `docs/research/`. Issue #16 closes.

## Open questions (flagged for future)

1. Do collaboration skills transfer across sessions as well as maintenance skills?
   (The v0.6.4 observation showed maintenance skills transferring cleanly. Collaboration
   skills depend on the human engaging with the prompt — untested.)
2. Is there a third skill category — *ecosystem* skills that coordinate across multiple
   human-agent pairs? `/bridge` hints at it but stays within one human's sessions.
3. How reliable are agent-side triggers for collaboration skills fired proactively
   (e.g., `/advocate` as a pre-ship check)? Maintenance-skill triggers are crisper.
