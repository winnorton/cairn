# Plan: v0.10 — `/resume` for Session-to-Session Handoff

**Status:** Shipped (validation pending)
**Date:** 2026-04-24
**Driving observation:** First link test of cairn's session handoff failed. New session in different worktree didn't auto-discover memory, HANDOFF.md, or transcript junction.

## Goal

Add a `/resume` skill that codifies the probe protocol for inheriting context from a
previous session. Defeat the namespace fragmentation that caused the link test to fail.

## Why

The 2026-04-24 link test was set up cleanly:
- Memory written to project memory (10 entries)
- `HANDOFF.md` at cairn repo root
- Transcript junction created at a known path
- cwar consolidation merged to main

Despite all of that, a fresh Claude Code session opened in a different cwar worktree
returned: *"No persistent memory exists yet, and there's no in-flight conversation
state to recall."* It then summarized cairn's GitHub state (which it inferred from
the repo + `gh`) but missed every cairn-side handoff artifact entirely.

Three reasons it failed:

1. **Memory is worktree-namespaced.** The cairn build session wrote to the *main cwar
   repo's* memory slug. The new session in `laughing-perlman-ff3b81` worktree looked at
   its own slug — a different directory.
2. **HANDOFF.md is convention-only.** Living at `cairn/HANDOFF.md` doesn't help when
   the new session is in cwar's worktree and has no reason to look at the cairn repo.
3. **Transcript junctions are ad-hoc.** The junction we set up for the Cairn Research
   workspace was a one-off; no agent knows to probe for similar links by default.

`/resume` is the named protocol that probes for all three.

## Design

### `/resume` skill

Triggers on: "resume", "where were we", "pick up", "continue from last session",
"what was happening", "what's the state."

Probes (in parallel) for:

1. **HANDOFF.md** at: `<cwd>`, `<cwd>/.claude`, `<cwd>/..`, `<cwd>/../..`, plus
   adjacent project roots if multi-repo setup.
2. **Memory directories** at multiple slugs: user-global, current-cwd-slug, parent-cwd-
   slug. **Probing the parent slug is the key fix for the worktree case.**
3. **Transcript junctions**: `cairn_link/`, `.previous_session/`, `session_link/`, or
   any directory that's a junction to a `.jsonl` or session dir.
4. **Git activity**: `git log -10 --oneline`, `git status --short`, `git stash list`,
   recent issues via `gh`.

Output: compact orientation in fixed format — what was happening / state at end /
anomalies / next action. Under ~250 words.

When nothing is found, lists every path probed (transparency for the user to point
at the right place manually).

### Tier classification

`role: scaffolding`, `tier: grow`. Universally applicable; not Claude-specific.
Pairs with `/reflect` (which produces the artifacts `/resume` reads).

### HANDOFF.md template note

Updated cairn's own `HANDOFF.md` to recommend `/resume` as the entry point for the
next session. If a session has cairn v0.10.0+ installed, `/resume` is the right move;
otherwise read HANDOFF.md directly.

## What was skipped

- **Auto-firing `/resume` at session start.** Would require harness changes (hooks)
  that aren't in cairn's scope. The user invokes it; the skill's description is sharp
  enough that capable agents will offer it when they detect prior-session signals.
- **Symlinking worktree memory to main project memory.** Would solve the namespace
  fragmentation more thoroughly but requires either user setup or a hook. Documented as
  an open issue for future work.
- **Cross-repo handoff** (e.g. cairn HANDOFF.md visible from cwar worktree) — `/resume`
  probes adjacent project roots but doesn't follow arbitrary repo links. Could be
  enhanced later with explicit cross-repo declaration in CLAUDE.md.

## Validation plan

After shipping v0.10.0:
1. Same setup as the 2026-04-24 link test: open a fresh Claude Code session in cwar's
   worktree.
2. User says "resume" or "where were we."
3. Expected: agent invokes `/resume`-style probing, finds HANDOFF.md at cairn repo OR
   memory at the parent slug, and produces orientation.
4. If still failing: iterate on the probe paths or document the manual workaround.

## Outcome

Shipped as v0.10.0. `/resume` skill, manifest update, HANDOFF.md cross-reference,
README + adopt.md preview updates. Tier file counts: seed=2, grow=9, structure=17,
full=18.

## Open questions for future work

- **Auto-fire `/resume` on session start** — can a Claude Code hook trigger this? Worth
  exploring once the manual invocation pattern proves out.
- **Cross-repo handoff** — should `CLAUDE.md` declare related repos (`## Related repos`)
  so `/resume` knows where else to probe?
- **Memory namespace as a first-class issue** — even with `/resume` patching the
  symptom, the underlying fragmentation (worktree memory != main memory) remains. A
  manifest-level "preferred memory slug" or a documented symlink pattern would address
  the root cause.
