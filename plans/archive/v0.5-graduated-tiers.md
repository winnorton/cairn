# Plan: v0.5 ‚Äî Graduated Tiers + Portability

**Status:** Shipped
**Date:** 2026-04-24
**Driving critique:** Issue #8 ‚Äî minimum viable habitat = maximally portable habitat (two files).

## Goal

Add a `tier` axis to the manifest alongside the existing `role` axis, so users can adopt
cairn at a graduated scope ‚Äî from the two-file agent-neutral core up through the full
Claude-ecosystem habitat. Make cross-agent portability an explicit design property rather
than an accidental one.

## Why

An in-habitat agent's research found that cross-session coherence requires only two files
(`CLAUDE.md` + `MEMORY.md`) ‚Äî and those happen to be the only files that are
ecosystem-neutral. Higher layers progressively add Claude-specific conventions:

- Skills use a Claude Code skill format (YAML frontmatter + description-based triggering).
- Typed-memory citations (`[MEM feedback/<name>]`) are a cairn convention.
- The `feedback` skill integrates specifically with cairn's own GitHub repo.

`role` labels (v0.4.3) answer "what matters most?" ‚Äî a reading-priority axis. But they
don't answer "how much cairn-specificity do you want?" ‚Äî the graduation/portability axis.
Two different axes, both worth exposing.

Cairn is <2 days old. Cheap structural time. User explicitly encouraged aggressive change.

## Design

**`tier` field added to every `files:` entry.** Values:

- **`seed`** ‚Äî `CLAUDE.md` + `memory/MEMORY.md`. Two files. Agent-neutral core.
- **`grow`** ‚Äî + `LAWS.md` + `skills/reflect` + `skills/plan`. Durable conventions any
  careful agent can follow.
- **`structure`** ‚Äî + memory type READMEs + `skills/{README,tour,prune,audit}`. Cairn's
  native hygiene loop.
- **`full`** ‚Äî + `skills/feedback`. Cairn-ecosystem integration.

Tiers are cumulative: `grow` includes `seed`; `structure` includes `grow`; `full` includes
all.

**Adoption invocation extended:**

- `adopt ‚Ä¶` ‚Üí default is `full` (unchanged behavior).
- `adopt ‚Ä¶ --tier seed` ‚Üí installs seed only (2 files).
- `adopt ‚Ä¶ --tier grow` ‚Üí installs seed + grow (5 files).
- `adopt ‚Ä¶ --tier structure` ‚Üí installs seed + grow + structure (13 files).
- `adopt ‚Ä¶ --tier full` ‚Üí installs all (14 files; explicit form of default).

Default stays `full` because decisions are expensive; users who want less ask for less.

**`manifest.tiers` section** defines each tier semantically. `adopt.md` Step 2 documents
how to interpret a `--tier <name>` flag in the adopt invocation.

**`role` and `tier` are orthogonal.** A file's `role` (essential/scaffolding/optional)
doesn't determine its `tier`. Example: `skills/feedback.md` is `optional` (role) but
`full` (tier) ‚Äî optional in the sense that capable agents file feedback anyway, but
full-tier because it integrates with cairn's own repo rather than being agent-portable.

## What was skipped

- No CLI wrapper for `--tier`. The flag is interpreted by the adopting agent reading
  the user's prompt, not by a binary. Keeps cairn dependency-free.
- No per-role-per-tier cross product ‚Äî users don't pick both dimensions independently at
  install time. Role is for reading priority post-install; tier is for install scope.
- No deprecation of the default behavior. `adopt ‚Ä¶` without `--tier` stays `full`.

## Outcome

Shipped as v0.5.0. Manifest adds `tier` field + `tiers` schema section. `adopt.md`
Step 2 documents tier filtering. README gains Portability + Graduated-adoption notes.
Issue #8 closes.
