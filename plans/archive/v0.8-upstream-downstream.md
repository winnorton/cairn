# Plan: v0.8 — Reflection Cadence + Upstream/Downstream Distillation

**Status:** Shipped
**Date:** 2026-04-24
**Driving critiques:** #14 (memory layer inactive during substantive work) and #17 (upstream/downstream distillate flow).

## Goal

Address two connected problems with one release:

1. **Memory writes aren't firing.** Fresh adoption produced 1,600+ lines of research and zero memory entries because `/reflect` never fired. The existing memory layer works when invoked; the failure is the invocation itself.
2. **Research doesn't flow between workspaces.** A research workspace (e.g. game engine research) produces findings meant to inform a build workspace (e.g. cwar-engine), but the habitat has no concept of "this workspace is upstream for that workspace" — so distillation for downstream habitats never happens.

Both problems share a root cause: **the reflection-and-distillation step isn't happening.** Fix that, and memory fires *and* flows to downstream consumers.

## Why

In the cairn build session, memory got written every time the agent reflected. In the cairn-test-3 (Game Engine Research) session, memory was empty because the agent never reflected. The skill works; it doesn't get called.

Encoding "reflect at natural checkpoints" as agent-default behavior — rather than waiting for the user to remember to ask — addresses the invocation gap. And extending `/reflect` to produce distillate for declared downstream consumers addresses the cross-workspace flow.

The operational pattern becomes: *research → reflect → distill → publish → downstream habitat inherits.*

## Design

### 1. Proactive reflection (addresses #14)

**`/reflect` skill** updated:
- Description now explicitly notes agent-proactive triggers (end of deep-dive, completed substantive chunk, milestone, before switching context).
- Steps section distinguishes user-invoked vs agent-proactive invocation.
- The proactive proposal is a single line: *"Natural reflection point — want me to run /reflect?"* User can decline; default is offering.

**New seed law #5 (Pause to reflect at natural checkpoints)** in `LAWS.md`:
- Encodes reflection cadence as habit, not only a user-invoked skill.
- Cites the root cause: reflection is what turns work into habitat.
- Pushes the seed law count from 5 to 6 — count references updated throughout.

### 2. Upstream/downstream distillation (addresses #17)

**`CLAUDE.md` template** gains optional `## Downstream consumers` section:
- Users declare which habitats this workspace's work feeds into.
- Format: one-line entries with consumer name, path, and one-line description of what applies.
- Commented out by default; removed entirely if not relevant.

**`/reflect` skill** — new step 4 "Check for downstream consumers":
- Reads `CLAUDE.md` for the downstream section.
- If present, additionally proposes distillate artifacts shaped **for the consumer's habitat**, not this workspace's.
- Distillate types: reference memory, project memory, law candidates, plan candidates.
- Language is in the consumer's vocabulary — *"per Unreal's GBuffer approach, cwar renderer should prefer X"* — not this workspace's framing.

**Step 6 writes distillate to `distillate/<consumer>/<type>/<slug>.md`** — predictable structure so the consumer habitat can ingest by folder walk. Transport (copy, symlink, git submodule) stays user-driven.

### 3. Skill triggers hierarchy documented

`/reflect` now has three trigger categories:
- **User-invoked (explicit):** "reflect", "take stock", affirmations to a reflect offer.
- **Agent-proactive:** natural checkpoints where the agent proposes, user approves.
- **Downstream-aware:** reflections in workspaces with declared consumers produce dual artifacts (local + distillate).

## What was skipped

- **Automatic sync of distillate to downstream habitats.** Transport stays user-driven. Automation is a separate concern (git submodules, symlinks, filesystem watchers) that shouldn't be baked into cairn's skill model.
- **Downstream agents pulling from upstream on demand.** Would require a network or symlink layer the user configures; not part of this release.
- **Validation that proposed distillate is actually applicable.** The agent proposes; the user judges per-entry. Same model as normal memory proposals.

## Outcome

Shipped as v0.8.0. Two issues close (#14 + #17). Reflection cadence is now habitual via law #5; downstream distillation is supported when declared. Count references updated (5 → 6 seed laws).

## Open questions (flagged for future)

- **Does proactive reflection feel intrusive in practice?** The proposal is opt-in per checkpoint, but even "want to reflect?" prompts have a friction cost. Watch for signal.
- **What's the right distillate transport mechanism at scale?** For one or two workspaces, manual copy is fine. For many, git submodules or a shared docs repo may be needed. Not solved here.
- **Can downstream agents PULL proactively, rather than depending on upstream pushing?** "My habitat knows cairn-research is my upstream; I should check for new distillate at session start." Future consideration.
