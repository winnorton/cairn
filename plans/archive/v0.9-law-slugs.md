# Plan: v0.9 — Slug-Based Law Identity

**Status:** Shipped
**Date:** 2026-04-24
**Driving critique:** Issue #18 — numeric law IDs drift and break citations; cwar's AGENTS.md is living evidence.

## Goal

Give cairn's laws stable identifiers (slugs) so citations survive reordering. Numbers
stay as display-order markers for human scanning, but they are no longer identity.

## Why

cwar's `AGENTS.md` provides the evidence — 14 current hard rules descended from 26 originals
through two renumbering passes in a **single day** (2026-04-22). The file carries a
renumbering tombstone because citations leaked outside the file:

- ESLint-disable comments in TypeScript files referencing rule numbers
- Cross-file references (`see Hard Rule 14`) in multiple CLAUDE.md descendants
- Commit messages
- Internal docs

Every renumber silently invalidates those. Cairn is on track to reproduce this pattern:
laws will accumulate citations in memory entries, plans, research docs, and the distillate
that flows downstream. Once they do, renumbering costs migration work (or breaks things
silently).

Memories in cairn already use slugs (`[MEM feedback/collaboration-style]`). Laws were the
outlier. Unifying the model around slugs resolves the inconsistency and preemptively
prevents the drift debt.

## Design

### Slug naming

Each seed law gets a short slug — kebab-case, 1-2 words, distinctive enough that it
doesn't clash with skill names:

| # | Law | Slug |
|---|---|---|
| 1 | Plan before executing anything hard to reverse | `plan` |
| 2 | Confirm before actions with blast radius beyond local sandbox | `confirm` |
| 3 | Surface assumptions | `assumptions` |
| 4 | Prefer editing existing artifacts to creating new ones | `prefer-edit` |
| 5 | Root-cause, don't route-around | `root-cause` |
| 6 | Pause to reflect at natural checkpoints | `cadence` |

`cadence` was chosen over `reflect` to avoid collision with the `/reflect` skill name.
`prefer-edit` rather than `edit` for distinctiveness (raw `edit` would be too generic).

### Changes

**LAWS.md template:**
- Each law header becomes `### N. <Rule> *(slug: <short-slug>)*`.
- Meta-rule 5 rewritten: slug-first citation, number form deprecated as brittle.
- `## Your laws` comment guides users to assign slugs when adding custom laws.

**`/audit` skill:**
- Regex updated: `\[LAW ([a-z][a-z0-9-]*)\]` (preferred) + `\[LAW (\d+)\]` (deprecated).
- When scanning, numeric citations resolve to slugs by reading LAWS.md headers.
- Output flags numeric citations as "candidates for migration to slug form."

**README + docs:**
- README citation table updated to show `[LAW plan]` / `[LAW cadence]` examples.
- Historical references in research docs LEFT UNCHANGED — they're frozen snapshots of
  observations at the time, not live citations. Updating them would rewrite history.

### Backwards compatibility

- `[LAW N]` citations remain parseable for transition. `/audit` counts them, but flags them.
- No migration required for existing `[LAW N]` citations in user habitats — the audit
  just pairs them with current slugs at scan time.
- Users (or their agents) can migrate existing numeric citations to slug form when
  convenient, but aren't forced to.

## What was skipped

- **Force-migrating numeric citations.** Users may have `[LAW 3]` in their own notes or
  downstream habitats. Audit surfaces these as candidates; migration is user-choice,
  not automatic.
- **Applying slugs to cwar's AGENTS.md rules.** Out of scope for cairn; a separate
  decision for the cwar project. The cwar data served as evidence for cairn's design;
  applying the pattern there is a cwar maintainer call.
- **Slugs for memory entries.** Memory entries already use slug-based paths
  (`feedback/collaboration-style`). No change needed.

## Outcome

Shipped as v0.9.0. Six seed laws gain slugs. Meta-rule 5 rewritten. `/audit` recognizes
both forms with slug preferred. Issue #18 closes.

## Principle worth naming

**When two parts of the same system use different identity conventions, the inconsistency
will eventually bite.** Memory had slugs; laws had numbers. That was a design accident,
not a deliberate choice — and it silently primed cairn to reproduce cwar's renumbering
pain. Fixing it required five minutes once spotted. Catching it required an active
analogical step: *laws are like memories; memories solved this; why haven't laws?*

This is the v0.9 lesson: **look across your own system for analogous problems and
borrow the solution.**
