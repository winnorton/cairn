The tone in skill docs (especially `/reframe`, `/bridge`, `/peer-review`) still defaults to *"agent defers to user."* **Candidate memory entry:** `files/memory/feedback/agent-evidence-over-human-recollection.md` codifying: in `/bridge`, `/peer-review`, `/reflect`, and especially `/round-review`, when the agent has tool-log evidence and the user has only recollection, **the agent surfaces the evidence first** before deferring; the user arbitrates with the evidence visible rather than relying on memory. Source: foundational session L2270+ (the Gemini correction episode), `NEW_LAWS Law 12` (the provenance line). 3-instance gate: currently **1/3** — file the memory entry when 2 more instances surface.

- **Why this isn't a cairn LAW:** the pattern is collaboration-style, not architectural. Belongs in feedback memory (mini-law shape, cited as `[MEM feedback/agent-evidence-over-human-recollection]`), not in `LAWS.md`.
- **Why it can wait:** the workaround is partially in place (`/bridge` Step 3.5 verifies claims before integrating). The remaining gap is *tonal* — skill docs still presume user-as-authority in ways that bias agents toward capitulating to human contradiction even when tool-log evidence is on the agent's side.
- **What the memory entry would say (draft):** *"When tool-log evidence and human recollection disagree, surface the evidence first; do not capitulate to the human's recollection without showing what the tool log actually says. The human can still override after seeing the evidence — that's arbitration. Capitulating without showing the evidence is deferral, not collaboration."*
